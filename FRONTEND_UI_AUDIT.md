## 1. Критические UI баги

### 🔴 1.1 XSS в CRMClientCard
**Файл:** `src/components/clients/CRMClientCard.tsx:56`
**Проблема:** `client.full_name` выводится без `sanitizeText()`, в то время как обычный `ClientCard.tsx` санитизирует. Если имя клиента содержит HTML — XSS.
**Решение:** Добавить `{sanitizeText(client.full_name)}`.

### 🔴 1.2 Формы без aria-label на всех input
**Файлы:** `/login`, `/register`, `/change-password`, `/consent`, все модалки
**Проблема:** Ни одно поле ввода не имеет `aria-label` или привязки через `aria-describedby`. Скринридеры не могут описать поля.
**Решение:** Добавить `aria-label` ко всем `<input>` / `<textarea>`.

### 🔴 1.3 Toast-уведомления невидимы для скринридеров
**Файл:** `src/layout/NotificationToast.tsx`
**Проблема:** Тосты не используют `aria-live="polite"`. Пользователи assistive technology не получают уведомлений.
**Решение:** Обернуть контейнер тостов в `<div role="status" aria-live="polite">`.

### 🔴 1.4 Модальные окна не ловят фокус (focus trap)
**Файлы:** Все модалки (BulkReassignModal, ClientCreateModal, InteractionCreateModal и т.д.)
**Проблема:** При открытии модалки Tab уходит «за» неё в основной контент. Нет focus trap.
**Решение:** Использовать `@radix-ui/react-focus-guard` или кастомный focus trap.

### 🔴 1.5 Анимации игнорируют prefers-reduced-motion в JS
**Файлы:** Параллакс на `/` (page.tsx), кристалл-микрофон, 3D-аватар
**Проблема:** CSS media query `prefers-reduced-motion` работает, но Framer Motion анимации продолжают работать в JS.
**Решение:** `const prefersReduced = useReducedMotion()` из framer-motion — и отключать animate.

## 2. Улучшения интерфейса

### 🟠 2.1 Skeleton loading вместо спиннеров
**Где:** `/home`, `/dashboard`, `/analytics`, `/history`, `/clients`, `/leaderboard`
**Сейчас:** Один `<Loader2>` спиннер с текстом «ЗАГРУЗКА...».
**Рекомендация:** Заменить на skeleton-лоадеры, повторяющие форму контента (карточки, графики, списки). Компонент `Skeleton.tsx` уже есть в `src/components/ui/`.
**Эффект:** Воспринимаемая скорость загрузки ×2-3.

### 🟠 2.2 Пустые состояния (empty states) на всех страницах
**Где:** `/home` (нет рекомендаций), `/history` (нет сессий), `/clients` (нет клиентов), `/analytics` (нет данных)
**Сейчас:** Только `/notifications` имеет empty state с иконкой Inbox.
**Рекомендация:** Единый паттерн empty state: иконка + текст + CTA-кнопка. Например: «Пока нет тренировок — [Начать первую]».

### 🟠 2.3 Переключатель видимости пароля
**Где:** `/login`, `/register`, `/change-password`
**Сейчас:** Нет возможности увидеть вводимый пароль.
**Рекомендация:** Иконка Eye/EyeOff в поле пароля. Особенно важно для мобильных.

### 🟠 2.4 Таймер сессии в тренировке
**Где:** `/training/[id]`
**Сейчас:** Длительность считается, но не отображается пользователю.
**Рекомендация:** Компактный таймер MM:SS в шапке сессии. С предупреждением при приближении к лимиту (MAX_SESSION_DURATION_MINUTES).

### 🟡 2.5 Индикатор силы пароля на всех формах
**Где:** `/change-password`
**Сейчас:** `/register` имеет индикатор (weak/medium/strong), `/change-password` — нет.
**Рекомендация:** Вынести `PasswordStrength` в переиспользуемый компонент.

### 🟡 2.6 Pagination на странице тренировок
**Где:** `/training`
**Сейчас:** Все сценарии загружаются сразу, без пагинации и поиска.
**Рекомендация:** Поиск по названию + пагинация (или infinite scroll).

### 🟡 2.7 Фильтрация и сортировка в истории
**Где:** `/history`
**Сейчас:** Простой список без возможности фильтрации.
**Рекомендация:** Фильтр по статусу (completed/abandoned), по баллу, по дате. Сортировка.

### 🟡 2.8 Breadcrumbs на вложенных страницах
**Где:** `/clients/[id]`, `/training/[id]`, `/results/[id]`, `/pvp/duel/[id]`
**Сейчас:** Нет навигационного контекста — пользователь не знает где он.
**Рекомендация:** Компонент `<Breadcrumbs>` с автоматическим построением по route.

### 🟢 2.9 Back button consistency
**Где:** Разные страницы используют разные паттерны «назад»
**Рекомендация:** Единый компонент `<BackButton>` с `router.back()` fallback на parent route.

### 🟢 2.10 Refresh кнопка на дашбордах
**Где:** `/dashboard`, `/analytics`, `/leaderboard`
**Сейчас:** Данные загружаются один раз, нет способа обновить без перезагрузки страницы.
**Рекомендация:** Иконка RefreshCw с ротацией при загрузке.

## 3. Дизайн-решения

### 🟠 3.1 Консистентность цветов: CSS vars vs inline vs Tailwind
**Проблема:** Три способа задания цветов используются одновременно:
- `style={{ color: "var(--accent)" }}` — inline CSS vars
- `className="text-purple-500"` — Tailwind classes
- `rgba(255,215,0)` — хардкод
**Рекомендация:** Миграция на единый подход — CSS custom properties через Tailwind (`text-[var(--accent)]` или extend в config). Хардкод цветов заменить на переменные.

### 🟠 3.2 Типографика: 3 шрифта, непоследовательное применение
**Сейчас:** Outfit (заголовки), Inter (текст), JetBrains Mono (код/метки). Но:
- Часть заголовков используют `.font-display`, часть — inline `fontFamily: "Outfit"`
- Размеры шрифтов: часть через Tailwind (`text-lg`), часть через классы из globals.css (`.t-page-title`)
**Рекомендация:** Утилитарные классы `.t-*` из globals.css — хороший паттерн, стоит применять его везде.

### 🟠 3.3 Border-radius: mixed values
**Сейчас:** `rounded-xl`, `rounded-2xl`, `rounded-[16px]`, `rounded-[22px]` — нет единого стандарта.
**Рекомендация:** Стандартизировать: `rounded-xl` (12px) для карточек, `rounded-2xl` (16px) для модалок, `rounded-full` для аватаров.

### 🟡 3.4 Кнопки: три CSS-класса + inline-варианты
**Сейчас:** `.vh-btn-primary`, `.vh-btn-outline`, `.vh-btn-danger` — хорошо, но часть кнопок используют inline styles для status colors.
**Рекомендация:** Вынести все варианты в CSS-классы (`.vh-btn-success`, `.vh-btn-warning`).

### 🟡 3.5 Opacity inconsistency
**Сейчас:** Фоновые overlay используют `0.1`, `0.08`, `0.15` без паттерна.
**Рекомендация:** Стандартизировать: `0.05` (subtle), `0.1` (light), `0.2` (medium), `0.5` (overlay).

## 4. Визуализация и детали

### 🟠 4.1 Score colors inaccessible для дальтоников
**Где:** `/results/[id]`, `/analytics`, `/dashboard`, `/leaderboard`
**Проблема:** Красный/зелёный — основные индикаторы хорошо/плохо. 8% мужчин с дейтеранопией не различают.
**Решение:** Добавить иконки (✓/✗/→) или паттерны к цветовым индикаторам. Или использовать оранжевый вместо красного.

### 🟠 4.2 Pentagram chart без alt-текста
**Файл:** `src/components/results/PentagramChart.tsx`
**Проблема:** SVG-радар без описания для скринридеров.
**Решение:** `<svg role="img" aria-label="Навыки: Скрипт 85%, Возражения 72%...">`

### 🟡 4.3 Emotion timeline: мелкие точки
**Файл:** `src/components/results/EmotionTimeline.tsx`
**Проблема:** На мобильных эмоциональные маркеры слишком мелкие для тапа (< 44px).
**Решение:** Увеличить touch target до 44×44px minimum (WCAG 2.5.5).

### 🟡 4.4 Dashboard: нет визуализации трендов
**Где:** `/dashboard`
**Сейчас:** Числа + стрелки up/down.
**Рекомендация:** Мини-графики (sparklines) для трендов за неделю рядом с каждой метрикой.

### 🟡 4.5 Pipeline kanban: нет счётчика суммы
**Где:** `/clients/pipeline`
**Сейчас:** Колонки показывают количество клиентов.
**Рекомендация:** Добавить сумму долга/выручки по колонке — key metric для РОПа.

### 🟢 4.6 Training session: нет прогресс-бара сессии
**Где:** `/training/[id]`
**Рекомендация:** Тонкий прогресс-бар вверху страницы, показывающий % от MAX_SESSION_DURATION.

### 🟢 4.7 Leaderboard: подсветка текущего пользователя
**Где:** `/leaderboard`
**Сейчас:** Нет выделения своей строки в таблице.
**Рекомендация:** Accent-border или glow на строке текущего пользователя.

## 5. WOW-моменты

### 🟢 5.1 Haptic feedback на мобильных (vibrate API)
**Где:** При поимке рыбы в FishermanError, при achievement unlock, при PvP match found.
**Реализация:** `navigator.vibrate?.(200)` — одна строка, сильный эффект.

### 🟢 5.2 Confetti на achievement unlock
**Где:** `/results/[id]` при получении бейджа.
**Сейчас:** Toast-уведомление.
**Рекомендация:** canvas-confetti (2KB) — визуальный взрыв конфетти при первом achievement.

### 🟢 5.3 Sound design расширение
**Где:** Все основные взаимодействия.
**Сейчас:** useSound хук есть, но используется мало (только gate open на лендинге).
**Рекомендация:**
- Мягкий «click» при отправке сообщения
- «Ding» при получении ответа AI
- Аплодисменты при score > 80%
- Тревога при TrapNotification

**Сейчас:** Мгновенная смена контента.

### 🟢 5.5 Easter egg в FishermanError
**Сейчас:** Уже WOW — рыбалка при ошибке сети.

**Где:** При входе в приложение, если streak > 3 дней.
**Рекомендация:** Анимация огня рядом со streak counter (уже есть StreakCounter компонент).

### 🟢 5.7 Dark mode transition ripple
**Сейчас:** Есть ripple-эффект в ThemeToggle.

## 7. Безопасность фронтенда

### 🔴 7.1 sanitizeText() inconsistency
**Проблема:** `CRMClientCard.tsx` не санитизирует `client.full_name`. ClientCard — санитизирует.
**Риск:** Stored XSS если имя клиента содержит `<script>`.
**Решение:** Унифицировать — все пользовательские данные через `sanitizeText()`.

### 🟠 7.2 dangerouslySetInnerHTML отсутствует ✓
Проверено 75+ компонентов — ни одного использования. Отлично.

### 🟠 7.3 CSP в next.config.ts
**Сейчас:** Production CSP включает `strict-dynamic`. Хорошо.
**Рекомендация:** Добавить `report-uri` или `report-to` для сбора CSP-нарушений.

### 🟡 7.4 Token в localStorage (fallback)
**Файл:** `src/lib/auth.ts`
**Проблема:** Хотя основной способ — httpOnly cookies, fallback хранит токен в localStorage.
**Риск:** XSS может прочитать localStorage.
**Рекомендация:** Убрать localStorage fallback, полностью на httpOnly cookies.

### 🟡 7.5 Нет rate-limit отображения на фронте
**Сейчас:** При 429 статусе нет user-friendly сообщения.
**Рекомендация:** Показывать «Слишком много запросов, подождите X секунд» с обратным отсчётом.

## 8. Accessibility

### Общая оценка: B (хорошая база, но много пробелов)

### Что есть ✓
- `aria-label` на ThemeToggle
- Семантический HTML в модалках (`<label htmlFor>`)
- `prefers-reduced-motion` в CSS (globals.css)
- Цветовые переменные для тем (light/dark)
- Touch-friendly размеры кнопок (44px+)

### Что отсутствует ✗

| Проблема | Приоритет | Затронуто |
|----------|-----------|-----------|
| `aria-label` на input полях | 🔴 | Все формы (login, register, change-password, modals) |
| `aria-live` на toast контейнере | 🔴 | NotificationToast |
| Focus trap в модалках | 🔴 | 8+ модальных окон |
| `aria-expanded` на dropdown | 🟠 | NotificationBell, фильтры на /clients |
| `aria-selected` на табах | 🟠 | /training, /pvp, /leaderboard, /notifications |
| `aria-describedby` для ошибок | 🟠 | Все формы с валидацией |
| Skip link (перейти к контенту) | 🟡 | Root layout |
| Keyboard navigation в dropdown | 🟡 | Все dropdown-меню |
| Alt-text на SVG иллюстрациях | 🟡 | Onboarding, FishermanError, Pentagram |
| `role="alert"` на error message | 🟡 | error.tsx, form errors |
| Reduced motion в Framer Motion JS | 🟡 | Параллакс, CrystalMic, Avatar3D |
