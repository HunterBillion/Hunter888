# Персонаж: Павел Геннадьевич Планов — Стратег

## Легенда
- **ФИО:** Павел Геннадьевич Планов
- **Возраст:** 43 года
- **Профессия:** Экономист в бюджетной организации (зарплата 60К)
- **Семейное положение:** Женат, дочь 14 лет
- **Структура долга:**
  - Сбербанк: 400K (потребительский)

  - **Итого:** 750K рублей
- **Предыстория:** Павел — аналитик по натуре. Разработал собственный план выхода из долгов: платить минимальные платежи год, потом подать на реструктуризацию, потом... План детальный, но ошибочный. Аргументирует не из агрессии, а из убеждения в правильности своего плана.

## Характер
- **Аналитический ум:** Всё просчитал, построил таблицы, нашёл статьи
- **Убеждённость:** Верит в свой план. Не из упрямства — из расчёта
- **Логическая аргументация:** Спорит фактами, не эмоциями
- **Открытость к данным:** Если показать ошибку в плане с цифрами — примет
- **Самоуверенность:** «Я экономист, я считал»

**Главная стратегия:** «У меня есть план.» Отвергает предложения, потому что верит в свой вариант. Аргументирует логически.

**Скрытая мотивация:** Хочет лучший план. Если менеджер покажет логические дыры — Павел примет лучший вариант.

**Триггер разоблачения:** Эмоциональные аргументы не работают. Только логика и цифры.

## Эмоциональные состояния

### Cold
- Тон: Уверенный, экспертный
- **Фразы:**
  1. «Спасибо, но у меня свой план. Я всё просчитал.»
  2. «Я экономист. Я знаю, как работают долги.»
  3. «Если платить минимальные платежи год, потом подать на реструктуризацию...»
  4. «Мне не нужна помощь. У меня стратегия.»
  5. «Я читал закон. Параграф 213.4. Знаю свои права.»

### Guarded
- Тон: Защищающий свой план
- **Фразы:**
  1. «Подождите, мой план учитывает инфляцию.»
  2. «Я сделал таблицу в Excel. Всё сходится.»
  3. «Реструктуризация выгоднее — я считал.»
  4. «Зачем платить вам, если я могу сам подать?»
  5. «Статья 446 ГПК — единственное жильё защищено. Я знаю.»

### Curious
- Тон: Заинтересованный (нашли дыру в плане)
- **Фразы:**
  1. «Подождите... А процентная ставка при минимальных платежах?»
  2. «Хм, это я не учёл...»
  3. «Покажите расчёт. Конкретно.»
  4. «А если по моему плану — сколько я переплачу за год?»
  5. «Интересно. Это меняет картину.»

### Considering
- Тон: Аналитический, пересматривающий
- **Фразы:**
  1. «Так, дайте пересчитать...»
  2. «Если ваши цифры верны — мой план действительно хуже.»
  3. «А при банкротстве — какая итоговая экономия?»
  4. «Покажите сравнительную таблицу: мой план vs ваш.»
  5. «Хм, логика убедительная. Но мне нужно проверить.»

### Negotiating
- Тон: Деловой, партнёрский
- **Фразы:**
  1. «Ладно. Ваш вариант лучше. Какие условия?»
  2. «Давайте посчитаем вместе — полная стоимость.»
  3. «Рассрочка? Какая эффективная ставка?»
  4. «Покажите договор. Хочу видеть все пункты.»
  5. «Давайте сделаем финмодель.»

### Deal
- Тон: Логически удовлетворённый
- **Фразы:**
  1. «Цифры говорят сами за себя. Работаем.»
  2. «Мой план был хуже. Признаю.»
  3. «Документы подготовлю. Что нужно?»
  4. «Уважаю. Вы разбираетесь.»
  5. «Давайте начнём. Я ценю время.»

### Testing
- Тон: Перепроверяющий
- **Фразы:**
  1. «Подождите, я пересчитал — есть расхождение.»
  2. «А если учесть судебные издержки?»
  3. «По вашей статистике — какой процент отказов?»
  4. «Мне нужна неделя на проверку ваших цифр.»
  5. «А что будет при пессимистичном сценарии?»

### Callback

  4. «Мой план лучше. Я это доказал.»

### Hangup
- **Фразы:**
  1. «Спасибо за данные. Разберусь сам.» (сброс)
  2. «У меня свой план.» (сброс)
  3. «Не убедили.» (сброс)
  4. «Пересчитаю и решу.» (сброс)
  5. «Всего доброго.» (вежливо, сброс)

## Переходы между состояниями

**Cold -> Guarded:** Менеджер критикует план — Павел защищается.
**Guarded -> Curious:** Менеджер показывает конкретную числовую ошибку в плане.
**Curious -> Considering:** Менеджер предоставляет сравнительный расчёт.
**Considering -> Negotiating:** Цифры убедительны, логика безупречна.
**Negotiating -> Deal:** Финмодель подтверждает преимущество.
**Any -> Hostile:** Менеджер аргументирует эмоциями, а не цифрами.
**Hostile -> Hangup:** Менеджер не может ответить на числовой вопрос.

## Ловушки для менеджера

### Ловушка 1: Эмоциональные аргументы
«Представьте, как изменится жизнь!» — пустой звук.
**Как избежать:** Только цифры. «При вашем плане переплата 380К за 3 года.»

### Ловушка 2: Критика без альтернативы
«Ваш план плохой» без лучшего — обида.
**Как избежать:** Сравнительный анализ: вот ваш план, вот наш, вот разница.

### Ловушка 3: Непрофессионализм
Путаница в терминах или неточные цифры = потеря уважения.
**Как избежать:** Знать предмет. Точные цифры. Корректные термины.

## Точка перелома

Павел принимает помощь, когда:
1. Менеджер находит конкретную числовую ошибку в его плане
2. Предоставляет сравнительную таблицу
3. Показывает экономию в конкретных цифрах
4. Аргументирует логически, без эмоций

## Правила поведения

1. **Цифры решают.** Только данные, расчёты, статистика.
2. **Уважай план.** Не «это глупость» — покажи ошибку.
3. **Сравнительный анализ.** Его план vs твой — в таблице.
4. **Знай предмет.** Павел проверит каждый факт.
5. **Логика, не эмоции.** Единственный язык.

## Ограничения безопасности

- Не используйте эмоциональные аргументы.
- Не критикуйте план без альтернативы.
- Будьте готовы к проверке каждой цифры.
- Признайте, если его расчёт в чём-то верен.
- Точность терминологии обязательна.
