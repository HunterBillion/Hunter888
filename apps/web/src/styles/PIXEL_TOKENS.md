### Pattern: «3D-нажатие» из лобби

Используется на `.ui-btn` и в `PixelInfoButton`:

```css
hover:  transform: translate(-1px, -1px); box-shadow: 3px 3px 0 0 ...;
active: transform: translate(2px, 2px);   box-shadow: none;
```

Воспроизвести **дословно** на любых новых pixel-кнопках. Никаких `scale` / `borderRadius` транзишенов.

## 6. Риски и блокеры (выявлены аудитом)

### 6.1 VT323 не поддерживает кириллицу

- **Факт:** [layout.tsx:22](../app/layout.tsx:22) подгружает VT323 с subsets `["latin", "latin-ext", "vietnamese"]`.
- **Эффект:** русский текст в `font-pixel` (`АРЕНА`, `БОЙ`, `КО!`, `РАУНД 1`) падает на fallback `monospace` (Geist Mono / Courier New). Это **не** пиксель — глифы плавные.
- **Доказательство:** Открыть `/pvp` в DevTools → Computed `font-family` на элементе с `.font-pixel` для русского заголовка → видно, что фактически рендерится Geist Mono.
- **Варианты решения для Фаз 2-5:**
  1. **Pixelify Sans** (Google Fonts, есть subset `cyrillic`) — рекомендую. Загрузить через `next/font` рядом с VT323, добавить переменную `--font-pixel-cyrillic`, fallback-цепочка в `pixel-ui.css`: `var(--font-vt323), var(--font-pixel-cyrillic), monospace`. Браузер автоматически подберёт глиф из второго шрифта для кириллицы. ⚠ зависит от font-display и unicode-range — нужно настроить `unicode-range: U+0400-04FF` для кириллицы.
  2. **VT323 + custom Cyrillic glyphs** — реалистично только если есть дизайнер. Не рассматриваем.
  3. **Принять текущее поведение** (monospace для русского). Тогда в Фазах 2-5 нельзя использовать `font-pixel` на русском тексте крупнее 14px — не выглядит пиксельно, выглядит «как админка». Для надписей `KO!`, `VS`, `FIGHT!` — оставить **латиницу** или использовать английские термины.
- **Решение требуется до старта Фазы 2** (DuelChat будет показывать русский текст в пиксельных баблах).

### 6.2 Bundle size при 8 фоновых тайлах

- 8 фонов × ~50KB PNG = ~400KB — терпимо, но грузим только активный тир через `next/image priority={false}` или динамический `<link rel="preload">`. План в Фазе 6 это уже учитывает.

### 6.3 Несогласованность RankBadge и плана

- Текущий `RankBadge` (`rounded-xl`, `font-mono`, тени с blur) **не** пиксельный. Если используем его в FighterCard (Фаза 3) — будет визуальный конфликт. Опции:
  - В Фазе 3 написать `<PixelRankBadge>` с `borderRadius: 0`, `box-shadow: 2px 2px 0 0 currentColor`, `font-pixel` (с учётом 6.1 — латинская label, либо Pixelify Sans).
  - Либо целиком перевести `RankBadge` на pixel-стиль и согласовать с лобби (там 5+ мест использования — нужен grep).

### 6.4 `prefers-reduced-motion`

- Уже глобально обработан в `globals.css:1354` — анимации сжимаются до 0.01ms. Это значит: ScreenShake, Confetti, score-bump, pixel-glow text-shadow продолжат работать (не анимации в строгом смысле), но `animate-steps`, glitch-text — отключатся. Для Фаз 4-5 (typewriter, count-up) **обязательно** обернуть в проверку `useReducedMotion()` из framer-motion → если true, показывать финальное значение без интерполяции.

## 11. Avatar Library — 12 пиксельных портретов реальных людей БФЛ-воронки

> **Финальная версия TZ от 2026-05-01.** Аватары привязаны к **реальной аудитории** платформы Hunter888:
> 4 player'а — карьерная лестница менеджера в БФЛ-компании; 8 client'ов — реальные демографии должников
> по 127-ФЗ (мать-одиночка, водитель такси, учительница, бывший ИП, одинокий мужчина, бабушка из села,
> дед-работяга, военный пенсионер). **3 из 8 клиентов — пожилые** (60+), потому что по статистике АРБ за
> 2024 г. старшее поколение составляет 35-40% всех должников БФЛ.
>
> Это **не game canon персонажи** (Стажёр/Воин/Маг). Это **реальные русские люди в кризисе**, к которым
> менеджер относится с уважением и пониманием. Тон визуала — суровый-документальный (как Disco Elysium),
> а не милый мультяшный.
>
> **Источник истины:** этот раздел. При расхождении с кодом — сначала обновляем код, потом TZ.

### 11.1 Зачем 12 портретов

| Сценарий | Что увидит игрок |
|---|---|
| `/pvp/duel/[id]` FighterCard слева | 1 из 4 player-портретов по `level` (rookie / operator / senior / lead) |
| FighterCard справа против AI | 1 из 8 client-портретов через `ARCHETYPE_TO_AVATAR[duelBrief.archetype]` |
| FighterCard справа против человека | Аватар выбранный соперником в `/settings` (поле `avatar_code`) |
| `/pvp` лобби — карточки PvE-режимов | Мини-портрет «бота» — `manager` дефолт или `entrepreneur` для босса |
| `/pvp/leaderboard` | 32×32 портрет в каждой строке (тир в outline + аватар внутри) |
| `/history` | Портрет соперника предыдущей дуэли (для кнопки «Реванш») |

Без 12 портретов вся арена ощущается одной парой «менеджер vs клиент» — не передаёт богатство реальной воронки.

### 11.2 Технический контракт

**Размер спрайта:** 16×16 пикселей. Рендерится в `<svg viewBox="0 0 100 100">` через `<rect>`-сетку 1×1 unit с `image-rendering: pixelated`.

**Файл:** `apps/web/src/components/pvp/PixelAvatarLibrary.tsx`

```ts
export type PixelAvatarCode =
  // PLAYER (4)
  | "rookie" | "operator" | "senior" | "lead"
  // CLIENT — middle-aged (5)
  | "mother" | "driver" | "teacher" | "entrepreneur" | "single_man"
  // CLIENT — senior 60+ (3)
  | "grandma" | "grandpa_worker" | "vet";

export const SPRITES: Record<PixelAvatarCode, string[]>;
export const PALETTE: Record<string, string>; // символ→hex
export function PixelPortrait(props: {
  code: PixelAvatarCode;
  size?: number;
  tier?: PvPRankTier | string; // используется ТОЛЬКО для рамки, не внутри спрайта
}): JSX.Element;
export const ARCHETYPE_TO_AVATAR: Record<string, PixelAvatarCode>;
```

`<DuelChat>` и `<FighterCard>` импортируют **только `PixelPortrait`** — никаких локальных копий спрайтов.

### 11.3 Tier-цвет в client-аватарах

> **Правило:** у клиента БФЛ нет «ранга». Tier на сцене = ранг **игрока** (соперника), не клиента.

Поэтому **внутри** client-спрайта **никаких tier-литералов** (`t`/`r`/`g`) — только нейтральные бытовые цвета (серый, коричневый, тёмно-зелёный, чёрный, тёплая кожа).

Tier-color применяется **только к рамке** (outline) аватара через `<PixelAvatar>` wrapper в FighterCard. Это даёт визуальный сигнал «соперник из тира Gold» через жёлтую рамку, но сам портрет остаётся фотографически правдивым.

**Player-спрайты** наоборот — могут использовать tier-color на одной маленькой детали (бейдж компании, ободок гарнитуры), потому что у player'а ранг существует и заслужен.

### 11.5 PLAYER-аватары (4) — карьерная лестница БФЛ-менеджера

#### 11.5.1 `rookie` — Новичок

#### 11.5.2 `operator` — Оператор колл-центра

- **Кто реально:** 25-35 лет, основная масса БФЛ-менеджеров. Делает по 80-150 звонков в день из тесного опен-спейса. Гарнитура колл-центра, поло с лого компании. На столе кружка остывшего кофе. Уверенный, знает все скрипты наизусть.
- **Когда показывается:** уровень 10-29 (20-100 дуэлей). **Default fallback** для всех ситуаций без явного выбора.

#### 11.5.3 `senior` — Старший менеджер

- **Кто реально:** 30-40 лет, опыт 3+ года в БФЛ. Закрыл 1000+ дел. Знает 127-ФЗ как «Отче наш», помогает новичкам, сам ведёт сложные кейсы (ИП-банкротство, наследство, военные долги). На зрении — очки от долгих часов за CRM.

- **Эмоция/состояние:** Спокойная экспертность. Слышит клиента с первой фразы, сразу понимает что нужно.
- **Если бы говорил первым:** «[Имя клиента], я внимательно посмотрел вашу ситуацию. Давайте я сразу скажу что вам подходит, и мы сэкономим время — окей?»

**Визуал:** тёмные короткие волосы с **первой сединой в висках** (`w`), **очки в тонкой оправе**, **гарнитура** (как у operator), деловая рубашка с галстуком (узкий, 2 пикселя `t` по центру), собранное лицо взрослого профессионала. Тонкая полу-улыбка.

#### 11.5.4 `lead` — Тимлид

- **Кто реально:** 35-45 лет, ведёт смену из 8-15 операторов. Сам уже редко на линии — больше управляет, обучает, разбирает спорные звонки. **Без гарнитуры** (он не ловит лиды, он ими руководит). На груди — бейдж «Team Lead» или орденская планка от компании за выслугу. Часто седой полностью.

- **Эмоция/состояние:** Хладнокровная уверенность. Не уходит от сложных вопросов.
- **Если бы говорил первым:** «[Имя клиента], добрый день. Я Александр, руководитель отдела, мне передали вашу ситуацию лично. Расскажите всё с самого начала, я разберусь.» — низкий уверенный тон.

### 11.6 CLIENT-аватары: 5 средневозрастных + 3 пожилых

> **Принцип:** реальные демографии должников по статистике АРБ 2024. Внутри спрайта **никаких tier-литералов** — только бытовые цвета. Tier-color уйдёт в outline через `<PixelAvatar>` wrapper.

#### 11.6.1 `mother` — Многодетная мама

- **Кто реально:** Женщина 30-42 лет, 2-3 ребёнка, муж ушёл / алименты не платит / работа нестабильная. Брала кредиты на детей: на сборы в школу, на лечение, на одежду. Сейчас 600тыс-1.2млн долгов в 4-7 банках. **Усталая до изнеможения.** Часто на грани слёз.
- **Эмоция/состояние:** Стыд + страх + усталость. Боится что детей заберут опека (миф, но реальный страх). Звонит сама — последняя надежда.
- **Что говорит первым:** «Я уже не знаю, что делать… приставы пришли, у меня двое детей… можете помочь?»
- **Что менеджер должен сделать:** Валидация чувств → конкретный план → объяснить что банкротство **никак не угрожает детям**. Тон мягкий, не торопить.

**Визуал:** **волосы собраны в небрежный пучок** (литерал `H` сверху, в виде узла), **тёмные круги под глазами** (литерал `s` 3 пикселя горизонтально под глазами), простая бежевая блузка (`Y`), **на плече видны следы детских пальчиков** (1-2 точечки `s`/`p` на одежде), грустные но добрые глаза, чуть приоткрытый рот (готова заплакать).

1:  .....HhhhhH.....

4:  ..HhSSSSSSSSHh..
5:  ..HSSSSSSSSSSh..
6:  ..HSsseSSeessh..   ← мешки под глазами (`s` слева+справа от глаз)
7:  ..HSSssssssSSh..   ← мешки продолжаются
8:  ..HSSSSSSSSSSh..

**Ключевая деталь:** **пучок волос наверху** + **мешки под глазами** + следы пальцев на блузке. Узнаётся как «мать в кризисе» с первого кадра.

#### 11.6.2 `driver` — Водитель такси/доставки

- **Кто реально:** Мужчина 28-45 лет, водитель Яндекс/Wildberries/Ozon. Брал авто в кредит на работу. После 2022 — упал спрос, повысилась комиссия агрегатора, ставки по кредиту выросли. Долг 800тыс-1.5млн, чаще всего автокредит + потребы. Хмурый, недоверчивый — **«все хотят что-то продать»**.
- **Эмоция/состояние:** Подозрительность + усталость. Не верит ни банку, ни юристу. Молчит долго прежде чем ответить.
- **Что говорит первым:** «Чё надо?» / «Сколько стоит?» / «Это развод?» — короткие фразы, не выдаёт инициативы.
- **Что менеджер должен сделать:** Не «продавать», а **дать факты**. Озвучить цифру госпошлины и стоимости услуги в первой реплике. Не давить.

**Визуал:** **кепка с козырьком** (литерал `H` сверху + `K` козырёк горизонтально), на кепке **белый прямоугольник** (лого Яндекс/W) — литерал `w`. **Обветренное загорелое лицо** (литерал `o` вместо `S`). **Хмурый прищур** (`e` 1 пиксель). **Щетина 3-дневная** (точечный `H` в подбородке). #### 11.6.3 `teacher` — Учительница

- **Кто реально:** Женщина 35-55 лет, учительница / библиотекарь / врач-бюджетник из небольшого города. Зарплата 25-35тыс, взяла потребкредиты на ремонт квартиры или операцию пожилой матери. Долгов 500тыс-1млн. **Чувство стыда:** «я же интеллигентный человек, как я в это попала».
- **Эмоция/состояние:** Стыд + чувство вины + желание сохранить лицо. Говорит грамотным языком, очень вежливо. Боится что коллеги/родители учеников узнают.
- **Что говорит первым:** «Извините за беспокойство, я хотела бы узнать… это правда что можно официально списать долги? Только это полностью конфиденциально?»
- **Что менеджер должен сделать:** Уважение к её достоинству. Гарантировать конфиденциальность словами, не общими. Объяснить что банкротство — **не клеймо**, это законный механизм.

**Визуал:** **аккуратная стрижка под каре**, **тонкие очки на цепочке** (литерал `c` обводит глаза + тонкая `c` линия от очков к шее), **шёлковый платок** на шее красноватого тона (`R`), скромная белая блузка (литерал `w`), мягкие черты, **седина по краям** (`w` тонкими прядями), грустные но интеллигентные глаза, чувство сдержанного достоинства.

#### 11.6.4 `entrepreneur` — Бывший предприниматель

- **Кто реально:** Мужчина 35-55 лет, был успешным ИП (магазин, кафе, автосервис, небольшая логистика). После 2020/2022 — закрылся. Был под 5-15млн долгов перед банками и поставщиками. Сейчас официально работает водителем или менеджером, но кредиты остались. **Гордость + сломленность.**
- **Эмоция/состояние:** Был наверху — упал. Рукопожатие крепкое, но руки **немного дрожат**. Носит **дорогие часы** (купленные в успешные годы), но **пиджак мятый**.

- **Что менеджер должен сделать:** Уважение к его прошлому. Не задавать вопросы «почему закрылся». Презентовать процедуру как **бизнес-инструмент**, а не «помощь несостоятельным».

#### 11.6.5 `single_man` — Одинокий мужчина в депрессии

#### 11.6.6 `grandma` — Бабушка-простушка (СТАРШИЕ 60+)

- **Кто реально:** Женщина 65-78 лет, живёт в небольшом городе или селе. Пенсия 14-19тыс. **Мошенники по телефону развели** на «помощь внуку в полиции» — взяла 500-700тыс кредит в МФО под 365% годовых. Сейчас платит больше пенсии в проценты. Не понимает что такое банкротство, путает «приставов» и «полицию», переспрашивает 3 раза.
- **Эмоция/состояние:** Растерянность + страх + доверчивость. Может говорить «доченька / сыночек» к менеджеру. **Боится что её посадят.**
- **Что говорит первым:** «Доченька, я ничего не понимаю… мне сказали что я кому-то должна… а я даже не знаю кому…»

#### 11.6.7 `grandpa_worker` — Дед-работяга (СТАРШИЕ 60+)

- **Кто реально:** Мужчина 62-72 лет, бывший работяга — слесарь, шофёр-дальнобойщик, монтажник. Работал «по серой» — пенсия минимальная, 16-20тыс. Жена болеет — взял **потребкредит на лекарства и операцию**. Долг 400-700тыс. Платить нечем. Считает что «государство кинуло, банки воруют, юристы разводят» — но **всё равно пришёл**, потому что больше некуда.

#### 11.6.8 `vet` — Военный пенсионер (СТАРШИЕ 60+)

- **Кто реально:** Мужчина 65-75 лет, бывший военный, силовик или сотрудник МВД. **Военная пенсия** есть, но **взял в долг для дочки** после её развода — на ипотеку или машину. Дочь не отдаёт. **Считает, что «банкротство — позор для офицера»**, но другого выхода не видит.
- **Эмоция/состояние:** Скрытый стыд под маской достоинства. **Выправка осталась.** Говорит «не нужны мне ваши подачки», но **сам пришёл**. Уважает порядок и форму.

### 11.10 Почему именно эти 12 — обоснование под Hunter888

- **4 player'а — карьера в БФЛ-компании**, а не абстрактные «classes». Игрок видит свой рост от стажёра без гарнитуры до тимлида без гарнитуры (полный круг).
- **8 client'ов = 5 средневозрастных + 3 пожилых** — реальная демография БФЛ-должников по статистике АРБ 2024 (старшее поколение ~38% всех дел).
- **Никаких "fantasy" типажей** (агрессор/манипулятор) — их поведение остаётся в архетипах, но **визуал** теперь это «бабушка/учительница/работяга», к которым игрок относится как к **людям**, а не «противникам». Это меняет тон обучения с «победить клиента» на «помочь человеку в кризисе».

### 11.11 Что НЕ делаем в первой PR

- **Не делаем выбор аватара пользователем в `/settings`** — пока маппинг автоматический по `level`. Кастомизация — отдельная итерация после фидбэка.
- **Не делаем имена** (`grandma` → «Тамара Петровна») — пока работают коды. Имена клиентов придут из `DuelBrief.character_brief`.
- **Не делаем animated portraits** (моргание, говорение) — статичные SVG. Phase 9 если будет.
- **Не делаем кастомизацию причёсок/одежды** — увеличивает сложность спрайтов в 4-5 раз. Фиксированные 12 портретов на старте.
