# ТЗ для художника: 50 3D-аватаров для Hunter888

**Версия:** 1.0
**Дата:** 2026-04-25
**Платформа:** [Hunter888](https://x-hunter.expert) — обучение продажам через ролевые диалоги с виртуальным клиентом
**Технология рендера:** [TalkingHead.js](https://github.com/met4citizen/TalkingHead) на базе Three.js
**Ожидаемый срок:** см. §10. Примерно 6-8 рабочих недель full-time.

---

## 0. TL;DR для художника

Нужно сделать **50 уникальных 3D-моделей-погрудных портретов (бюст + голова)** в формате **GLB**. Каждая модель — отдельный архетип клиента-должника (см. §5).

Каждая модель должна:
1. Иметь **Mixamo-совместимый скелет** (это критично — без него анимации движка не работают).
2. Содержать **52 ARKit blendshapes** (мимика — улыбка, удивление, гнев и т.д.).
3. Содержать **15 Oculus Visemes** (формы рта при произнесении звуков).
4. Загружаться в браузере через TalkingHead.js без ошибок.
5. Весить ≤ 10 MB после оптимизации (Draco compression).

**Pipeline:** MPFB Blender extension → Blender → glTF Export → optimize → проверка через мою страницу-валидатор → передача мне.

## 1. Контекст проекта

**Hunter888** — симулятор холодных звонков для коллекторов и юристов по банкротству. Менеджер звонит виртуальному «клиенту» (нашему AI), отрабатывает скрипт, получает оценку. Наш AI-клиент — это:
- голос (ElevenLabs/Azure TTS на русском)
- эмоциональное состояние (10 эмоций: cold/hostile/curious/considering/negotiating/deal/...)
- **3D-аватар, который ты делаешь** — даёт визуальное правдоподобие звонка

**Откуда берутся 50 архетипов:** из реального файла кода `apps/web/src/lib/archetypes.ts` — там 100 архетипов в 10 группах по поведенческим паттернам. Я отобрал 50 самых низко-тирных (которые игроки видят чаще всего). Полный список — §5.

**Где аватар появляется в UI:**
- **Главное место**: режим звонка (`/training/[id]/call`) — большой портрет в центре экрана, говорит, мимикрирует, реагирует на эмоциональное состояние диалога
- **Чат-режим**: маленький портрет рядом с сообщениями
- **Превью в конструкторе**: при сборке клиента в Lego-builder'е игрок видит как изменяется внешность

## 2. Технические требования (КРИТИЧНО)

### 2.1. Формат файла

- **Расширение:** `.glb` (binary glTF 2.0)
- **НЕ принимаются:** `.fbx`, `.obj`, `.vrm`, `.blend`
- **Размер файла:** идеально 4-8 MB, максимум 10 MB
- **Compression:** обязательно Draco mesh compression (включается при экспорте из Blender)
- **Текстуры:** PNG, embedded в GLB (не внешние файлы), 1024×1024 или 2048×2048 максимум
- **Формат текстур внутри GLB:** PNG (НЕ JPG — на лице видны JPG-артефакты)

### 2.2. Скелет (rig) — критично

Скелет ОБЯЗАН быть **Mixamo-совместимым**. Это значит — стандартные имена костей с префиксом `mixamorig`:

```
mixamorigHips           ← root
mixamorigSpine
mixamorigSpine1
mixamorigSpine2
mixamorigNeck
mixamorigHead           ← голова (обязательно для движений головы)
mixamorigLeftEye        ← левый глаз
mixamorigRightEye       ← правый глаз
mixamorigLeftShoulder
mixamorigRightShoulder
mixamorigLeftArm        ← Bicep
mixamorigRightArm
mixamorigLeftForeArm
mixamorigRightForeArm
mixamorigLeftHand
mixamorigRightHand
mixamorigLeftHandThumb1..3
mixamorigLeftHandIndex1..3
... (все пальцы — для жестов)
```

**Как получить:** в MPFB при экспорте выбрать rig type = **"Game engine"** или **"Standard rig"**, имена костей переименовать через скрипт (я предоставлю rename-скрипт для Blender).

**Если делаешь без MPFB:** используй [Mixamo Auto-Rigger](https://www.mixamo.com/) — загружаешь модель, Mixamo автоматически делает rig + переименовывает кости. Это самый простой способ.

### 2.3. Морф-таргеты (blendshapes) — ОБЯЗАТЕЛЬНЫЕ

Без правильных morph targets аватар не сможет ни говорить, ни выражать эмоции.

#### 2.3.2. Oculus 15 visemes (формы рта при речи)

Эти морф-таргеты движок дёргает каждый кадр когда аватар говорит:

```
sil   ← рот закрыт, нейтральная позиция
PP    ← смыкание губ (звуки п/б/м)
FF    ← верхние зубы касаются нижней губы (ф/в)
TH    ← язык между зубами (англ. th — для русского почти не используется, но обязателен)
DD    ← язык за верхними зубами (т/д/н/л)
kk    ← задняя часть языка поднята (к/г/х)
CH    ← округлые губы, язык поднят (ч/щ/ш/ж)
SS    ← узкая щель между зубами (с/з/ц)
nn    ← как DD но с резонансом носа (н/л в русском)
RR    ← дрожащий звук (р)
aa    ← широко открытый рот (а/я)
E     ← полуоткрытый, растянутый (э/е)
I     ← узкий, растянутый (и/ы/й)
O     ← округлый средний (о/ё)
U     ← округлый узкий (у/ю)
```

**Критично:** имена morph-таргетов точно в этих регистрах (например `aa` маленькими, `PP` заглавными).

#### 2.3.3. Дополнительные morph targets (рекомендуемые)

```
mouthOpen           ← общее открытие рта (используется как fallback)
mouthSmile          ← улыбка (общая)
eyesClosed          ← закрытые глаза
eyesLookUp / eyesLookDown
```

### 2.4. Масштаб и ориентация

- **Высота головы (от подбородка до макушки):** ~0.22 единиц Blender = ~22 см в реальном мире
- **Hips Y-координата:** 0 (модель центрирована на hip)
- **Forward axis:** −Z (модель смотрит в −Z направлении после rotateVRM0)
- **Up axis:** +Y

### 2.5. Topology

- **Quad-based** (четырёхугольники) — не triangulate перед экспортом, glTF сам сделает это
- **Polygon count:**
  - Голова: 8000-15000 полигонов (для хороших морфов нужны качественные треугольники в районе рта/глаз)
  - Бюст (плечи + торс до пояса): 3000-6000 полигонов
  - Итого на модель: 12000-20000 полигонов
- **Smooth shading** включено
- **UV map** один (uv_0), без оверлапов в районе лица

### 2.6. Materials

- 1 материал на голову/тело (PBR: BaseColor + Normal + Roughness + Metallic)
- Опционально: отдельный материал на одежду
- **Standard glTF PBR shader** (НЕ кастомные shader nodes — экспорт их проигнорирует)
- Прозрачность: только в волосах если нужно (alpha clip)

## 3. Naming convention

Каждый файл именуется по `archetype_code` из таблицы §5:

```
<archetype_code>.glb
```

Примеры:
- `skeptic.glb` (Скептик)
- `family_man.glb` (Семьянин)
- `lawyer_client.glb` (Юрист-клиент)

**НЕ** именовать файлы кириллицей или с пробелами.

### 4.3. Перед сдачей — ОБЯЗАТЕЛЬНО

1. **Проверить через [Babylon.js Sandbox](https://sandbox.babylonjs.com/)** — drag&drop GLB, убедиться что модель загружается без ошибок и все morph targets перечислены в правой панели.
2. **Проверить через мою страницу-валидатор** — я предоставлю URL после старта работы. Художник дропает GLB → видит зелёные/красные галочки по каждому из 67 обязательных morph targets (52 ARKit + 15 Oculus).
3. **Тест в TalkingHead** — я предоставлю минимальную HTML-страницу для тестирования lipsync на конкретной модели.

## 6. Этническая и стилистическая палитра

**Целевая аудитория Hunter888:** русскоязычные коллекторы → клиенты-должники в России. Поэтому:

- **Этничность:** преимущественно европейская внешность (славянская). Допустимо разнообразие: 2-3 модели кавказской внешности, 1-2 азиатской — для реалистичности (РФ — многонациональная).
- **Стиль:** реалистичный (НЕ stylized cartoon, НЕ аниме, НЕ Pixar). Похоже на Ready Player Me realistic preset или Avaturn.
- **Эмоциональная палитра:** аватары не "красивые модели" — это уставшие, обычные люди в трудной финансовой ситуации. Морщины, усталость, обычные лица. Никаких супермоделей.

**Референсы стилистики (mood board):**
- [Avaturn realistic avatars](https://avaturn.me/) — основной референс
- [Synthesia AI avatars](https://www.synthesia.io/) — для деловых типажей
- Не использовать как референс: VRoid, MetaHuman (слишком hi-poly)

## 10. Сроки и оценка работы

## 11. Что НЕ делаем (явные исключения)

- ❌ Полное тело ниже пояса — нам нужны только бюст + голова (камера всегда показывает upper body)
- ❌ Анимации (walk, run, idle, gesture) — за это отвечает TalkingHead движок, ему нужны только morph targets, скелет он анимирует сам
- ❌ Hair physics / cloth physics — отключено, рассчитывается мимикой
- ❌ Scenes / lighting — фон рендерит Three.js на нашей стороне, аватар отдаётся на прозрачном фоне
- ❌ Высокополигональные MetaHuman-style модели — таргет ≤ 20k полигонов
