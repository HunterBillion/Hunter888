# P0 — Микрофон в звонке: подключается долго и тихо ломается

**Статус:** open, backlog L0
**Зафиксировано:** 2026-04-29 со слов product owner
**Приоритет:** P0 (блокирует пилот — пользователи не могут говорить с AI)

## Симптомы

1. **Подключение очень медленное.** Пользователь входит в сессию → микрофон долго стартует → ощущение что «ничего не работает».
2. **Тихий отказ.** Если у пользователя нет разрешения на микрофон / не подключён микрофон / включён mute на устройстве — система **никак не сигнализирует**. Пользователь говорит, но AI не реагирует. Не понимает почему.
3. **Полный сбой.** Часть пользователей вообще не получает работающего микрофона — STT не приходит на бэк, сессия выглядит «зависшей».

## Возможные корни (требуют диагностики, не решений)

| Кандидат | Где смотреть |
|---|---|
| Web Speech API permission prompt не виден / отказ tihio | `apps/web/src/hooks/useSpeechRecognition.ts` |
| Deepgram streaming WS не подключается до Accept-click | `app/services/stt_deepgram.py` + Deepgram-config |
| Whisper batch fallback не активируется | `app/services/stt.py` + `whisper` контейнер |
| HTTPS требуется — пользователь на http:// | nginx config + browser |
| iOS Safari getUserMedia — отдельный quirk | `useSpeechRecognition` browser detection |
| Микрофон unlock ждёт первого user-gesture | `apps/web/src/app/training/[id]/call/page.tsx` |

## Требования к фиксу (User-first)

1. **Видимый прогресс при подключении** — спиннер «подключаем микрофон…» с таймаутом
2. **Явная ошибка при отказе доступа** — toast «Браузер не дал доступ к микрофону. Что делать → гайд»
3. **Pre-flight check ДО начала сессии** — кнопка «проверь микрофон» на /training-list, не давать стартовать сессию пока mic не подтверждён
4. **Fallback на текстовый ввод сразу** — если mic не работает, поле ввода уже доступно
5. **Логирование на бэк** — сколько % сессий стартует без mic, чтобы был метрик-сигнал

## Что НЕ делать

- Большой рефактор STT pipeline. Только UX-обвязку: статус, ошибки, fallback
- Отключать Deepgram / Whisper в пользу одного — сейчас многослойность правильная
- Менять backend контракт — это чисто FE задача
